## üéØ Usage Guide

### **For Frontend Developers:**

**Starting a New Screen?**
1. Open the screen file (e.g., `/screens/WeatherScreen.tsx`)
2. Read the annotation block at the top (starts with `/**`)
3. Follow the conversion requirements
4. Use code examples from `/COMPLETE_CONVERSION_EXAMPLE.md`
5. Check testing checklist before marking complete

**Building a Component?**
1. Open the component file (e.g., `/components/TrendCard.tsx`)
2. Read the annotation block at the top
3. Follow React Native conversion notes
4. Test on both iOS and Android

### **For Backend Developers:**

**Setting Up APIs?**
1. Read: [`BACKEND_DATA_REQUIREMENTS.md`](./BACKEND_DATA_REQUIREMENTS.md)
2. Find your feature's section
3. Implement the documented endpoints
4. Use the request/response shapes provided
5. Integrate external APIs as documented

### **For Project Managers:**

**Planning Sprints?**
1. Check: [`SCREEN_CONVERSION_STATUS.md`](./SCREEN_CONVERSION_STATUS.md)
2. Use priority ratings for task ordering
3. Refer to time estimates in [`ANNOTATION_COMPLETION_STATUS.md`](./ANNOTATION_COMPLETION_STATUS.md)
4. Follow recommended implementation order

### **For QA/Testers:**

**Testing a Feature?**
1. Open the screen file
2. Scroll to bottom of annotation
3. Use the "Testing Checklist" section
4. Test all items on both iOS and Android
